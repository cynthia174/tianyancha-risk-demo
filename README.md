# 天眼查企业合作风险等级 Demo

输入企业简称或全称后，从天眼查数据中选择精确主体，并输出高、中、低三级企业合作风险。当前规则版本为 v3.0，不再使用百分制。

## 主要能力

- 企业简称模糊搜索和候选企业下拉选择
- 高、中、低三级风险结论，不使用百分制
- 电话、邮箱等疑似关系达到 20 条直接判定高风险
- 被执行人、失信、限高、股权冻结、开庭公告和裁判文书逐项解释
- 行政处罚、经营异常、严重违法、欠税、破产和注销风险逐项解释
- 法人关联企业可点击跳转到对应天眼查页面
- 法人关联企业存在周边高风险信号时判中风险；异常密集且大量注销时判高风险
- 法人周边高风险会标出具体关联公司、主要风险及可点击链接
- 社保 0 人、住宅、公寓、单间和小办公室仅作核验提示，不改变总体等级
- 历史名称无论数量多少均仅作核验提示，不改变总体等级
- 成立不足 1 年判中风险；失信、限高、破产、异常主体状态等红线判高风险
- 规则列表按当前企业的风险等级和业务重要性动态排序
- 每条规则展示判定原因和原始历史证据
- 总体等级下方自动生成按重要性组织的风险结论说明
- 一键跳转天眼查企业原始页面

## 本地运行

要求 Node.js 18+，并已安装、初始化 `tyc-cli`。

```bash
npm install -g tyc-cli
tyc init --authorization "你的天眼查 API Key"
npm start
```

打开 `http://127.0.0.1:4173`。

## Docker 部署

```bash
docker build -t tyc-risk-demo .
docker run --rm -p 4173:4173 -e TYC_API_KEY="你的密钥" tyc-risk-demo
```

不要把密钥写入代码、Dockerfile 或 GitHub 仓库。部署平台应通过 Secret/Environment Variables 注入 `TYC_API_KEY`。

## GitHub 部署说明

GitHub Pages 只能托管静态文件，无法安全运行本项目的 Node 后端，也不能保存天眼查 API Key。因此本仓库提供 Docker 配置，建议从 GitHub 仓库连接 Render、Railway、Fly.io 或其他容器平台，并在平台中设置 `TYC_API_KEY`。GitHub 仓库本身用于版本管理和自动部署源代码，不在 Pages 中暴露密钥。

## 免责声明

风险等级仅用于商业合作初筛，不构成征信、法律或投资结论。公开数据可能存在延迟或缺失，重大合作应进行人工尽调。
